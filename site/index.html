Tutorial

添加教程

  1. 先取得 fakeid。登录微信公众平台，在图文编辑器的“超链接 / 选择其他公众号文章”中搜索目标公众号；打开浏览器开发者工具的 Network，找到名称含 searchbiz 的请求，在响应中搜索 fakeid 并复制。
  2. 不要填 gh_ 开头的 user_name。本服务需要公众平台搜索接口返回的 Base64 风格 fakeid，通常类似 MzA...==。
  3. 在左侧生成配置。公众号名称和 fakeid 必填，微信号与备注可以不填；重复名称或 fakeid 会被阻止。
  4. 提交到 GitHub。点击“复制 JSON”后打开 GitHub 编辑页，全选替换 config/accounts.json，点击 Commit changes 提交。
  5. 让配置生效。等待次日 10:00，或前往 Actions 手动运行 Daily WeChat job scan ↗，将 reprocess_hours 保持为 0。
  6. 检查结果。任务成功后，网页公众号数量会增加；若显示账号失败，优先检查 fakeid 是否复制完整、公众号是否可在公众平台搜索。
为什么不能在网页直接保存？

GitHub Pages 是公开静态网页。把仓库写入令牌放进网页会导致任何访客都能修改配置，因此必须在已登录 GitHub 的编辑页完成最后提交。

自动筛选仅用于减少信息噪声，岗位要求请以公众号原文与招聘单位官网为准。
